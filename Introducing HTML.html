What is Html?

Html is a Programing language used for web development.

It stand for HYPER-TEXT MARKUP LANGUAGE

Syntax

There are some rules you have to follow while writing HTML.
These set of rules are called Syntax of HTML.

  1. The (!DOCTYPE html) declaration should always be the first line of code in your HTML files. This lets the browser know what version of HTML to expect.
  2. The (html) element will contain all of your HTML code.
  3. You can add a title to your web page by using the (title) element, inside of the head. A webpage’s title appears in a browser’s tab.
  4. The title tag is written in head.
  5. Html consists of a many tags for writing codes.
    • Almost every tag has an opening and closing tag.
    • The opening tag is simply in two comparison symbols. While a closing tag has a slash(/) in the starting after the smaller than symbol.
    • But some tags are self closing tags in which they only have one opening tag.
      In those self closing tag a slash(/) is put before the greater than symbol.

Links

Links help us to directly go to specific places or websites through just a click.

  1. The Anchor tag is used for Links.
  2. There is a href attribute which is in the anchor tag. We Enter the URL there.
  3. While linking to websites which are stored in the same folder in which the script is can be done by typing just this in the href.(./NameOfDocument.html)
  4. An image or video can be made as a link by make the img tag a child of anchor tag.
  5. To link to the same page on certain location,you can create divs there with an id and type (#nameOfId) in the href.
  6. We can set a (target) attribute in the anchor tag and set it to (_blank) for opening the link in a separate tab.

Tables

The (table) element creates a table in which we enter data in tabular form.


  1. The (tr) element adds rows to a table.
  2. To add data to a row, you can use the (td) element.
  3. Table headings clarify the meaning of data. Headings are added with the (th) element.
  4. Table data can span rows using the rowspan attribute.
  5. Table data can span columns using the colspan attribute.
  6. The scope attribute can be added to table heading for defining the scope.(Row or column)
  7. Tables can be split into three main sections: a head, a body, and a footer.
    • A table’s head is created with the (thead) element.
    • A table’s body is created with the (tbody) element.
    • A table’s footer is created with the (tfoot) element.

Forms

The purpose of a (form) is to allow users to input information and send it.
  1. The (form)‘s action attribute determines where the form’s information goes.
  2. The (form)‘s method attribute determines how the information is sent and processed.
  3. To add fields for users to input information we use the (input) element and set the type attribute to a field of our choosing:
    • Setting type to "text" creates a single row field for text input.
    • Setting type to "password" creates a single row field that censors text input.
    • Setting type to "number" creates a single row field for number input.
    • Setting type to "range" creates a slider to select from a range of numbers (We can set the value by min and max attributes.).
    • Setting type to "checkbox" creates a single checkbox which can be paired with other checkboxes.
    • Setting type to "radio" creates a radio button that can be paired with other radio buttons.
    • Setting type to "list" will pair the (input) with a (datalist) element if the id of both are the same.
    • Setting type to "submit" creates a submit button.
  4. A (select) element is populated with (option) elements and renders a dropdown list selection.
  5. A (datalist) element is populated with (option) elements and works with an (input) to search through choices.
  6. A (textarea) element is a text input field that has a customizable area.
  7. When a (form) is submitted, the name of the fields that accept input and the value of those fields are sent as name=value pairs.
  8. The (option) tag is used to create a value.(An attribute value is added.)
  9. The (label) tag is used in forms and linked to the (input) tag through (for) and (id) attributes.
  10. The (datalist) tag and (input) tag are linked by (id) and (list) attributes.
  11. The(select) and (label) tags are linked through (id) and (for) attributes.
  12. The (required) attribute in the (input) tag will make it necessary for a user to fill the input.
  13. Assigning a value to the (min) attribute of a number input element will validate an acceptable minimum value.
  14. Assigning a value to the (max) attribute of a number input element will validate an acceptable maximum value.
  15. Assigning a value to the (minlength) attribute of a text input element will validate an acceptable minimum number of characters.
  16. Assigning a value to the (maxlength) attribute of a text input element will validate an acceptable maximum number of characters.
  17. Assigning a regex to (pattern) matches the input to the provided (regex).
  18. The (regex) is written as( [a-zA-Z0-9]{12-15} ).In this it is defining that the pattern can be any number or letter with minimum 12 digits and maximum 15 digits.
  19. The (step) attribute is used to define the speed of scrolling.(In range)
    • We can add the step attribute in (input type no.) for choosing the number through scrolling.

  20. If validations on a (form) do not pass, the user gets a message explaining why and the (form) cannot be submitted.

Semantic HTML

Semantic HTML introduces meaning to a page through specific elements that provide context as to what is in between the tags. Semantic HTML is a modern standard and makes a website accessible for people who use screen readers to translate the webpage and improves your website’s SEO.

  • (header), (nav) , (main) and (footer) create the basic structure of the webpage.
  • (section) defines elements in a document, such as chapters, headings, or any other area of the document with the same theme.
  • (article) holds content that makes sense on its own such as articles, blogs, comments, etc.
  • (aside) contains information that is related to the main content, but not required in order to understand the dominant information.
  • (figure) encapsulates all types of media.
  • (figcaption) is used to describe the media in (figure).
